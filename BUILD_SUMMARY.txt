================================================================================
ISOLATION GUARANTEES - 10 IMMUTABLE LAWS
================================================================================

 1. ✅ Isolation is Mandatory
    Every agent runs in fully isolated sandbox; no escape possible

 2. ✅ Memory is Dual
    Separate STM (volatile) and LTM (persistent) memory systems

 3. ✅ Tools are Fungible
    Stateless tool invocations, no direct API access

 4. ✅ Execution is Auditable
    Complete immutable audit trail of all actions

 5. ✅ State is Recoverable
    Crash recovery via checkpoints and WAL

 6. ✅ Security is Layered
    Namespaces + cgroups + seccomp + capabilities + MAC

 7. ✅ Resources are Bounded
    Hard limits on memory, CPU, files, processes (enforced by kernel)

 8. ✅ Policy is External
    Security rules defined outside code, can be updated without recompile

 9. ✅ Transparency is Intrinsic
    All decisions, patterns, options are queryable

10. ✅ Failure is Fast
    Security violations halt execution immediately; no silent failures

================================================================================
AGENT EXECUTION
================================================================================

ReAct Loop:
  1. Planning - Analyze prompt, select next tool
  2. Execution - Invoke tool, capture result
  3. Reflection - Update memory, log decision
  Repeat until: max iterations reached or task complete

Tool Selection Strategies:
  - Keyword matching (read/write/run/list)
  - Pattern history (learned success sequences)
  - Exploration (occasionally try new tools)
  - Fallback (echo tool as safe default)

Multi-Agent:
  - Coordinator for managing agent swarms
  - Parent-child sandbox hierarchy
  - Inter-agent messaging via channels
  - Isolated execution contexts

Prompting Techniques:
  - Chain-of-Thought (step-by-step reasoning)
  - Few-Shot (examples + task)
  - ReAct (reasoning + acting with reflection)
  - Error recovery with exponential backoff
